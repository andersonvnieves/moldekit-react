# Icons

This design system uses the Lucide icon set via lucide-react for consistent SVG icons across the application.
Icons are React components that are tree‑shaken, configurable by props, and render inline SVGs.

## Guidelines

### When to use

Use icons to clarify actions, status, and object types. Prefer pairing icons with text labels in navigation and buttons to improve recognition and accessibility of meaning.

### Sizing and color

Lucide defaults are `size = 24`, `color = currentColor`, and `strokeWidth = 2`, which map well to common UI scales and typographic systems.
Using the CSS/SVG keyword currentColor ensures the icon inherits the parent text color, keeping icons consistent with themed foregrounds across light and dark modes.

### Accessibility

Decorative icons should be hidden from assistive tech with aria-hidden="true" so they don’t pollute accessible names or cause redundancy for screen readers.
Icons that convey meaning or act as controls need an accessible name via aria-label or visible text; avoid relying on color alone to communicate state or status.

## Usage

### Import and basics

Import icons directly as components, use props like `size`, `color`, and `strokeWidth`.

> Tip: Centralize defaults with a small wrapper that inherits text color and uses `aria-hidden` for decorative usage.

### Dynamic import

> Tip: Only use the DynamicIcon variant when icon names come from runtime data such as a CMS or configuration.

```js
    import {DynamicIcon} from "lucide-react/dynamic";
```

```jsx
    <DynamicIcon name="chevron-left" />
```

### Static import

> Tip: Only use the DynamicIcon variant when icon names come from runtime data such as a CMS or configuration.

```js
    import { Camera } from "lucide-react";
```

```jsx
    <Camera strokeWidth={5} color={'#FAFAFA'} />
```
