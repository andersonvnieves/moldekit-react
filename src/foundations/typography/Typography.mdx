# Typography

The MoldeKit design system’s typography scale establishes a consistent, accessible hierarchy using **two specialized font families**:
Inter for text content andRoboto Mono for numeric values.
Built with Tailwind utilities, these semantic CSS classes map directly to HTML elements while ensuring data alignment and readability.

## Dual Font Strategy

MoldeKit uses a deliberate **dual-font approach** for optimal readability and precision:

- **`Inter`** for headings, body text, buttons, labels and caption. Clean, variable-width glyphs optimized for extended reading
- **`Roboto Mono`** for numbers, prices, metrics, and tabular data. Fixed-width characters prevent visual distortion in columns and ensure perfect decimal alignment

**Why two fonts?** Variable-width fonts like Inter create beautiful prose but misalign numbers.
Monospace fonts solve this for data tables, forms, and financial displays while maintaining design consistency.

## Accessibility

Semantic HTML elements provide built-in accessibility roles for screen readers, eliminating the need for manual ARIA attributes on typography classes.
Use correct heading tags and content elements.
These announce hierarchy and structure automatically to assistive technologies like NVDA or VoiceOver.

## Title

Page/app titles, hero sections, landing pages, main navigation headers

> Reserve  mk-title  for page-level headers only, each page should have only one element with this class applied.

### Sample:

<Typeset
  fontFamily="var(--font-sans)"
  fontSizes={["32px"]}
  fontWeight={500}
  sampleText={<h1 className="mk-title">Title</h1>}
/>

### Usage

Apply `mk-title` on HTML text elements

```tsx
<main>
  <h1 className="mk-title">Title</h1>
</main>
```
